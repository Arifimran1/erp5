Commandments

\n

Opinionated Guidelines for Developing Web Applications.

\n
\n
\n Email\n Download\n
\n
\n
\n \n
\n
\n
\n
\n
\n
\n
\n
\n

1. Smallest Size First

\n

\n - Applications should support multiple devices (smartphone, tablet,\n desktop, ...) but be designed for smallest size first -\n

\n
\n \n
\n \n
\n
\n
\n
\n
\n

2. Single Codebase

\n

\n - A single codebase should tailor to all devices/platforms -\n

\n
\n \n
\n \n
\n
\n
\n
\n
\n

3. Interaction over Typing

\n

\n - The smaller the device the more cumbersome typing is.\n Enforce interactions vs requiring users to type -\n

\n
\n \n
\n \n
\n
\n
\n
\n
\n

4. Usability over Functionality

\n

\n - Ease of use dictatetes application design, not functionalities\n provided -\n

\n
\n \n
\n \n
\n
\n
\n
\n
\n

5. Embrace Path of Least Interaction

\n

\n - The less interactions are necessary to complete an action, the better -\n

\n
\n \n
\n \n
\n
\n
\n
\n
\n

6. Embrace Interaction Flow

\n

\n - Interaction elements are positioned within the "flow" of\n application usage (eg. form save after form) -\n

\n
\n \n
\n \n
\n
\n
\n
\n
\n

7. Key Interactions Visible without Scrolling

\n

\n - The key interactions of a page are accessible without the user\n having to scroll -\n

\n
\n \n
\n \n
\n
\n
\n
\n
\n

8. Highlight Key Interaction

\n

\n - The key interaction is highlighted for ease of navigation -\n

\n
\n \n
\n \n
\n
\n
\n
\n
\n

9. Animate Only If Possible

\n

\n - Use animations only if they work. Partially working animations ruin look & feel!! -\n

\n
\n \n
\n \n
\n
\n
\n
\n
\n
\n
\n

10. Four Themes: Light + Dark + Color + Active

\n

\n - Each theme with a main and light color (total eight colors) -\n

\n
\n \n
\n \n
\n
\n
\n
\n
\n
\n
\n

11. One Font, Four Sizes, Two Weights

\n

\n - Applications use a single font in up to four sizes and two weights - \n

\n
\n \n
\n \n
\n
\n

12. Max Two breakpoints per device

\n

\n - One or two breakpoints per device (smartphone, tablet, desktop, ...)\n must suffice -\n

\n
\n \n
\n \n
\n
\n

13. Flat Single Color Icons

\n

\n - Iconsets must be single color, flat icons -\n

\n
